# [Прогнозирование заказов такси](taxi_demand_forecasting.ipynb)

# Краткое описание

Проект посвящён прогнозированию количества заказов такси для компании «Чётенькое такси». Целью исследования являлось построение модели машинного обучения, способной предсказывать число заказов на следующий час, чтобы компания могла привлекать необходимое количество водителей в периоды повышенного спроса.

В работе использовались исторические данные о заказах такси в аэропортах. В ходе проекта были выполнены подготовка временного ряда, ресемплирование данных по часам, исследование тренда и сезонности, проверка стационарности ряда, создание признаков на основе временных лагов и скользящего среднего, обучение и сравнение нескольких моделей машинного обучения, а также оценка качества лучшей модели на тестовой выборке.

# Цель проекта

Построить модель машинного обучения для прогнозирования количества заказов такси на следующий час с метрикой RMSE не более 48.

# Задачи проекта

* Загрузить и изучить данные временного ряда.
* Проверить хронологический порядок наблюдений.

* Исследовать тренд временного ряда.
* Исследовать суточную сезонность.
* Исследовать недельную сезонность.
* Проверить стационарность временного ряда.
* Сформировать новые признаки:

  * календарные признаки;
  * лаговые признаки;
  * признаки скользящего среднего.
* Разделить данные на обучающую и тестовую выборки.
* Обучить и настроить модели машинного обучения:

  * LinearRegression;
  * DecisionTreeRegressor;
  * RandomForestRegressor;
  * CatBoostRegressor;
  * LGBMRegressor.
* Выполнить подбор гиперпараметров моделей.
* Оценить качество моделей по метрике RMSE.
* Выбрать лучшую модель.
* Проверить качество модели на тестовой выборке.
* Выполнить проверку модели на адекватность.

## Основные результаты

### Результаты анализа временного ряда

* Выявлен устойчивый положительный тренд роста количества заказов такси.
* Установлена выраженная суточная сезонность:

  * максимальное количество заказов наблюдается около полуночи;
  * минимальное количество заказов наблюдается около 6 часов утра.
* Выявлена недельная сезонность:

  * пики заказов приходятся на понедельник и пятницу;
  * минимальные значения наблюдаются во вторник.
* По результатам теста Дики-Фуллера сделан вывод о стационарности временного ряда.

### Результаты моделей

| Модель                |    RMSE |
| --------------------- | ------: |
| LinearRegression      | 22.7488 |
| DecisionTreeRegressor | 29.2400 |
| RandomForestRegressor | 24.7278 |
| CatBoostRegressor     | 22.5034 |
| LGBMRegressor         | 23.2650 |

Лучшие параметры моделей:

**LinearRegression**

* fit_intercept = False

**DecisionTreeRegressor**

* max_depth = 9
* max_features = 5
* min_samples_leaf = 8
* min_samples_split = 12

**RandomForestRegressor**

* n_estimators = 300
* max_depth = 15
* max_features = 10
* min_samples_leaf = 12
* min_samples_split = 16

**CatBoostRegressor**

* depth = 5
* iterations = 700

**LGBMRegressor**

* num_iterations = 800
* max_depth = 5
* learning_rate = 0.1

### Итоговый результат

* Лучшее качество на обучающей выборке показала модель **CatBoostRegressor** с RMSE **22.5034**.
* На тестовой выборке модель **CatBoostRegressor** показала RMSE **37.2548**.
* Полученное значение удовлетворяет требованию проекта (**RMSE < 48**).

### Проверка на адекватность

* RMSE наивной модели по предыдущему значению составило **58.8565**.
* RMSE модели CatBoostRegressor составило **37.2548**.
* Модель машинного обучения показала более высокое качество прогнозирования по сравнению с наивным подходом.

## Вывод

Разработанная модель позволяет прогнозировать количество заказов такси на следующий час и может использоваться для планирования нагрузки сервиса. Благодаря прогнозу компания получает возможность заранее привлекать необходимое количество водителей в периоды повышенного спроса и снижать риск дефицита транспорта.

Наилучшие результаты показала модель CatBoostRegressor, достигнув значения RMSE **37.25** на тестовой выборке, что соответствует требованиям проекта. Проверка на адекватность подтвердила, что модель существенно превосходит простой прогноз по предыдущему значению и может быть использована для поддержки операционных решений.
