# [Рекомендация тарифов мобильной связи](ML_intro.ipynb)

# Краткое описание

Проект посвящён построению модели машинного обучения для рекомендации тарифного плана мобильной связи на основе данных о поведении пользователей. В работе использовался датасет `users_behavior.csv`, содержащий информацию о количестве звонков, длительности разговоров, числе сообщений, объёме интернет-трафика и признаке принадлежности пользователя к тарифу Ultra.

В ходе проекта были выполнены загрузка и проверка данных, подготовка признаков, разделение выборки на обучающую, валидационную и тестовую части, обучение и сравнение нескольких алгоритмов классификации. Для выбора лучшей модели проводился подбор гиперпараметров и оценка качества по метрике Accuracy. Дополнительно была выполнена проверка модели на адекватность с использованием DummyClassifier.

# Цель проекта

Построить модель классификации, способную рекомендовать пользователю подходящий тарифный план на основе характеристик его поведения.

# Задачи проекта

* Загрузить и изучить данные.
* Проверить данные на наличие пропусков и дубликатов.
* Привести типы данных к корректному формату.
* Подготовить признаки и целевой признак для обучения моделей.
* Разделить данные на обучающую, валидационную и тестовую выборки в соотношении 60% / 20% / 20%.
* Обучить и сравнить модели классификации:

  * DecisionTreeClassifier;
  * RandomForestClassifier;
  * LogisticRegression.
* Подобрать гиперпараметры для каждой модели.
* Оценить качество моделей на валидационной выборке по метрике Accuracy.
* Выбрать лучшую модель.
* Проверить качество лучшей модели на тестовой выборке.
* Проверить модель на адекватность с помощью DummyClassifier.
* Сформулировать итоговые выводы по результатам исследования.

# Итоги

## Выполненные задачи

* Провела загрузку и первичное изучение данных.
* Выполнила проверку данных на пропуски и дубликаты.
* Привела столбцы `calls` и `messages` к целочисленному типу данных.
* Разделила данные на обучающую, валидационную и тестовую выборки в пропорции 60% / 20% / 20%.
* Обучила модели DecisionTreeClassifier, RandomForestClassifier и LogisticRegression.
* Выполнила подбор гиперпараметров для каждой модели.
* Рассчитала значение Accuracy для всех исследуемых моделей.
* Проверила качество лучшей модели на тестовой выборке.
* Проверила адекватность модели с помощью DummyClassifier.

## Основные результаты

* В данных не обнаружены пропуски и дубликаты.
* Для модели DecisionTreeClassifier получено максимальное значение Accuracy **0.7776049766718507** при параметрах:

  * `min_samples_split = 2`;
  * `min_samples_leaf = 1`;
  * `max_leaf_nodes = 9`;
  * `max_depth = 7`.
* Для модели RandomForestClassifier получено максимальное значение Accuracy **0.7993779160186625** при параметрах:

  * `max_features = None`;
  * `bootstrap = True`;
  * `max_depth = 9`;
  * `n_estimators = 10`.
* Для модели LogisticRegression получено максимальное значение Accuracy **0.7262830482115086** при параметрах:

  * `solver = 'newton-cg'`;
  * `max_iter = 100`.
* Наилучший результат на валидационной выборке показала модель RandomForestClassifier с Accuracy **0.7993779160186625**.
* Accuracy модели RandomForestClassifier на тестовой выборке составила **0.7931570762052877**.
* Accuracy модели DummyClassifier составила **0.6889580093312597**.

## Вывод

Полученная модель позволяет автоматически рекомендовать пользователям подходящий тарифный план на основе их фактического поведения. Это может снизить затраты на ручной анализ клиентских данных и повысить качество персонализированных предложений.

Наиболее эффективной оказалась модель RandomForestClassifier, показавшая Accuracy около 0.80 на валидационной выборке и 0.79 на тестовой. Результаты проверки на адекватность подтвердили, что модель работает существенно лучше простого базового алгоритма и может использоваться для решения задачи рекомендации тарифов.
