# [Защита персональных данных клиентов](linear_algebra.ipynb)

# Краткое описание

Проект посвящён защите персональных данных клиентов страховой компании с помощью преобразования признаков методом умножения на обратимую матрицу. Основная задача заключалась в разработке способа шифрования данных, который позволяет скрыть исходную информацию о клиентах без ухудшения качества модели машинного обучения.

В работе использовались данные страховой компании, содержащие сведения о поле, возрасте, заработной плате, количестве членов семьи и числе страховых выплат. В ходе проекта были выполнены загрузка и предобработка данных, теоретическое доказательство корректности преобразования признаков, реализация алгоритма шифрования, обучение моделей линейной регрессии на исходных и преобразованных данных, а также проверка возможности восстановления исходных данных.

# Цель проекта

Разработать метод преобразования персональных данных клиентов, который обеспечивает их защиту без ухудшения качества модели линейной регрессии.

# Задачи проекта

* Загрузить и изучить данные страховой компании.
* Проверить данные на наличие пропусков.
* Проверить данные на наличие дубликатов.
* Привести типы данных к корректному формату.
* Определить целевой и признаковые столбцы.
* Теоретически доказать влияние умножения признаков на обратимую матрицу на качество линейной регрессии.
* Вывести формулу преобразования весов модели после шифрования данных.
* Разработать алгоритм шифрования признаков.
* Сгенерировать обратимую квадратную матрицу.
* Выполнить преобразование признаков путём умножения на обратимую матрицу.
* Обучить модель линейной регрессии на исходных данных.
* Обучить модель линейной регрессии на зашифрованных данных.
* Сравнить качество моделей на исходных и преобразованных данных.
* Проверить возможность восстановления исходных данных с помощью обратной матрицы.
* Сделать вывод о применимости предложенного метода защиты данных.

## Выполненные задачи

* Провела загрузку и изучение данных страховой компании.
* Выполнила проверку данных на пропуски и дубликаты.
* Привела признаки возраста и заработной платы к целочисленному типу данных.
* Удалила обнаруженные дубликаты.

* Вывела зависимость новых весов модели от исходных весов после преобразования данных.

* Оценила корректность предложенного метода защиты данных.

## Основные результаты

* В данных отсутствовали пропуски.

* Обнаруженные дубликаты были удалены.

* Доказано, что при умножении матрицы признаков на обратимую матрицу качество линейной регрессии не изменяется.

* Получено аналитическое выражение для новых весов модели:

  * `w₁ = P⁻¹w`

* Разработан алгоритм шифрования данных, основанный на умножении матрицы признаков на случайную обратимую матрицу размером 4×4.

* Проверка алгоритма показала, что качество модели на исходных и зашифрованных данных остаётся одинаковым.

* Для обеих моделей получено значение метрики:

  * **R² = 0.43**

* Подтверждена возможность восстановления исходных данных путём умножения зашифрованной матрицы признаков на обратную матрицу.


## Вывод

Предложенный метод позволяет защитить персональные данные клиентов без потери качества модели машинного обучения. После преобразования исходные значения признаков становятся недоступны для прямого анализа, при этом модель линейной регрессии сохраняет прежнюю точность прогнозирования.

Результаты проекта показывают, что матричное преобразование может использоваться как механизм обезличивания данных при передаче информации между подразделениями компании или внешними исполнителями. Такой подход обеспечивает дополнительную защиту персональных данных и одновременно сохраняет возможность построения и использования моделей машинного обучения.
