# [Отток клиентов банка](supervised_learning.ipynb)

# Краткое описание

В проекте решалась задача прогнозирования оттока клиентов банка. На основе исторических данных о клиентах необходимо было построить модель, способную предсказывать вероятность ухода клиента из банка. Особое внимание уделялось проблеме дисбаланса классов, поскольку доля ушедших клиентов значительно меньше доли оставшихся.

Для анализа использовались данные о клиентах банка, содержащие информацию о кредитном рейтинге, возрасте, стране проживания, балансе счёта, количестве продуктов банка, активности клиента и других характеристиках. В ходе работы были выполнены подготовка данных, кодирование категориальных признаков, масштабирование количественных признаков, исследование баланса классов, обучение и сравнение нескольких моделей классификации, а также применение методов борьбы с дисбалансом классов.

# Цель проекта

Построить модель машинного обучения для прогнозирования оттока клиентов банка с достижением целевого значения F1-меры не менее 0.59.

# Задачи проекта

* Загрузить и изучить исходные данные.
* Проверить структуру данных и типы признаков.
* Удалить неинформативные признаки:

  * RowNumber;
  * CustomerId;
  * Surname.
* Обработать пропуски в данных:

  * заполнить пропуски в признаке Tenure средним значением.
* Проверить данные на наличие дубликатов.
* Разделить данные на обучающую, валидационную и тестовую выборки.
* Подготовить категориальные признаки:

  * выполнить кодирование методом One-Hot Encoding.
* Подготовить количественные признаки:

  * выполнить стандартизацию признаков.
* Исследовать баланс классов целевого признака.
* Обучить модели на несбалансированных данных:

  * дерево решений;
  * случайный лес;
  * логистическая регрессия.
* Подобрать гиперпараметры моделей.
* Оценить качество моделей по метрикам F1 и ROC-AUC.
* Выполнить балансировку классов:

  * методом upsampling;
  * методом downsampling;
  * с использованием параметра class_weight='balanced'.
* Сравнить результаты различных способов балансировки.
* Проверить лучшие модели на тестовой выборке.
* Выбрать итоговую модель для прогнозирования оттока клиентов.

## Основные результаты

* Баланс классов в исходных данных составил:

  * 79.63% клиентов остались в банке;
  * 20.37% клиентов ушли из банка.

* На несбалансированных данных были получены следующие результаты:

  * Дерево решений:

    * F1 = 0.5971428571428572;
    * ROC-AUC = 0.8380352617640755.
  * Случайный лес:

    * F1 = 0.5952380952380952;
    * ROC-AUC = 0.8485110688500518.
  * Логистическая регрессия:

    * F1 = 0.3214953271028037;
    * ROC-AUC = 0.7875656858707707.

* После балансировки классов были получены следующие результаты для случайного леса:

  * Upsampling:

    * F1 = 0.6358511837655016;
    * ROC-AUC = 0.8610798780290305.
  * Downsampling:

    * F1 = 0.6079545454545454;
    * ROC-AUC = 0.8596524104998682.
  * class_weight='balanced':

    * F1 = 0.6384039900249376;
    * ROC-AUC = 0.8598105038783005.

* На тестовой выборке модель случайного леса с class_weight='balanced' показала:

  * F1 = 0.604591836734694;
  * ROC-AUC = 0.4793653437721235.

* На тестовой выборке модель случайного леса, обученная на данных после upsampling, показала:

  * F1 = 0.5983889528193326;
  * ROC-AUC = 0.7560079339740357.

* Лучшими результатами по ROC-AUC на тестовой выборке обладала модель случайного леса, обученная на данных после upsampling:

  * max_depth = 18;
  * n_estimators = 20;
  * max_features = 2;
  * min_samples_leaf = 5.

* Целевое значение F1-меры было достигнуто.

## Вывод

Построенная модель позволяет прогнозировать вероятность ухода клиентов банка и выявлять клиентов с повышенным риском оттока. Использование такой модели помогает заранее принимать меры по удержанию клиентов и снижать потери, связанные с их уходом.

Проведённое сравнение методов балансировки показало, что учёт дисбаланса классов существенно повышает качество прогнозирования. Полученные результаты могут использоваться для построения системы раннего предупреждения оттока клиентов и более эффективного распределения маркетинговых и сервисных ресурсов банка.
